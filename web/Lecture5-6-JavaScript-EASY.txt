ğŸ“Œ SECTION 14: PROTOTYPE - BETTER WAY FOR METHODS

ğŸ”¹ THE PROBLEM WITH CONSTRUCTORS:
   Every time you create a new object, the functions are DUPLICATED in memory.
   
ğŸ”¹ THE SOLUTION - PROTOTYPE:
   Define methods on the PROTOTYPE so all objects SHARE them.

   function Item(title) {
     this.title = title;
     this.status = "available";
   }

   // Methods on PROTOTYPE (shared by all Item objects!)
   Item.prototype.isAvailable = function() {
     return this.status == "available";
   };

   Item.prototype.issue = function() {
     this.status = "issued";
   };

   var item1 = new Item("HTML Spec");
   var item2 = new Item("JS Spec");
   // Both share the SAME isAvailable and issue functions!
   // More memory efficient!

ğŸ“Œ SECTION 16: INHERITANCE (extends)
================================================================================

ğŸ”¹ WHAT IS INHERITANCE?
   - One class can INHERIT from another (get all its stuff)
   - Child class gets parent's properties and methods
   - Child can ADD its own stuff or OVERRIDE parent's

   // Parent class
   class Animal {
     constructor(name) {
       this.name = name;
     }

     speak() {
       return `${this.name} makes a sound`;
     }
   }

   // Child class EXTENDS parent
   class Dog extends Animal {
     constructor(name, breed) {
       super(name);            // Call parent's constructor!
       this.breed = breed;     // Add new property
     }

     // Override parent's method
     speak() {
       return `${this.name} barks!`;
     }
   }

   const dog = new Dog("Rex", "German Shepherd");
   console.log(dog.name);      // "Rex" (inherited from Animal)
   console.log(dog.breed);     // "German Shepherd" (Dog's own)
   console.log(dog.speak());   // "Rex barks!" (overridden method)

   // super() = call parent class constructor
   // extends = inherit from parent class

   // Inside, it called add(5, 3) which returned 8
   // Then printed "Result: 8"
