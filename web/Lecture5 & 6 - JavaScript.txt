JavaScript’s Reputation
• Everything is type sensitive, including function, class, and variable names.
• The scope of variables in blocks is not supported. This means variables declared inside a loop may be accessible outside of the loop, counter to what one would expect.
• There is a === operator, which tests not only for equality but type equivalence.
• Null and undefined are two distinctly different states for a variable. Null Means no value / empty value intentionally. undefined means Variable declared but not initialized
• Semicolons are not required, but are permitted (and encouraged).

Arithmetic operators
JavaScript performs implicit type coercion
String Coercion (Number → String) 
When a number is added to a string, JavaScript converts the number into a string and performs concatenation. console.log(2 + "2"); // "22"
Number Coercion (String → Number)
When a string contains a number and is used with mathematical operators (-, *, /), JavaScript converts it into a number. console.log("10" - 5); // 5

Objects

function Item(t){

  this.title = t;
  this.status = "available";
  this.isAvailable = function (){
	return this.status == "available"; 
  }

}

alert(item1.isAvailable()); // true

JS Loops
for (expression 1; expression 2; expression 3) {
  // code block to be executed
}
Expression 1 is executed (one time) before the execution of the code block.
Expression 2 defines the condition for executing the code block.
Expression 3 is executed (every time) after the code block has been executed.
